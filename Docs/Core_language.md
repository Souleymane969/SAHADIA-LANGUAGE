## Core Language Definition (v1.0)

This document defines the **core (native) features** of SAHADIA LANGUAGE.  
These features are **built into the language itself** and **do not belong to any library**.

> The core language is minimal, stable, and eternal.

---

## 1. CORE PHILOSOPHY

SAHADIA LANGUAGE is designed to be:

- Human-readable
- Symbol-free
- Emotion-aware
- Beginner-friendly
- Forgiving by design

The core language must always remain:
- simple
- stable
- minimal
- expressive

## 6. CORE RULES

1. One line equals one intention
2. No mandatory symbols
3. No explicit typing
4. Errors should be gentle
5. Meaning over performance

---

## 7. WHAT DOES NOT BELONG TO THE CORE

The following features **must not** be part of the core language:

- Advanced mathematics
- String manipulation helpers
- Date & time utilities
- Random generation
- Web frameworks
- Game engines
- Artificial intelligence systems

These features belong to the **Standard Library** or external libraries.

---

## 8. CORE STABILITY GUARANTEE

Once a feature is part of the core language:
- it should not be removed
- it should not change meaning
- it should remain backward compatible

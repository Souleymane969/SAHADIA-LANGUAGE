# Chapitre 7  
## Retourner des valeurs — Comprendre le résultat  
*Return, fonctions utiles, calculs propres*

Dans le chapitre précédent, nous avons appris à nommer des actions.
Ces actions pouvaient afficher des messages ou effectuer des calculs.

Mais parfois, nous ne voulons pas seulement agir.
Nous voulons **obtenir un résultat**.

C’est le rôle du retour de valeur.

---

## 1. Pourquoi retourner une valeur ?

Une fonction qui retourne une valeur permet de dire :

> *Fais ce travail, puis donne-moi le résultat.*

Ce résultat peut ensuite être :
- affiché,
- stocké,
- réutilisé,
- transformé.

Sans retour de valeur,
les fonctions restent limitées.

---

## 2. La philosophie du retour en SAHADIA

Dans SAHADIA :
- une fonction peut **produire une valeur**
- cette valeur est **clairement indiquée**
- rien n’est implicite

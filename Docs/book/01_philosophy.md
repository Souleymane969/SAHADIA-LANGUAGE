# Chapitre 1  
## Philosophie de SAHADIA LANGUAGE

### 1. Le langage comme acte humain

Avant d’être une suite de règles,
un langage est une manière de penser.

La plupart des langages de programmation ont été conçus
pour parler à des machines,
puis adaptés pour être tolérables aux humains.

SAHADIA fait l’inverse.

Il parle d’abord à l’humain,
et seulement ensuite à la machine.

Chaque construction du langage repose sur une question simple :
> *Est-ce que cette idée peut être comprise sans effort inutile ?*

Si la réponse est non, alors la construction est repensée,
ou simplement refusée.

---

### 2. La clarté avant la performance

SAHADIA ne cherche pas à être le plus rapide.
Il cherche à être le plus clair possible.

La performance est importante.
Mais une performance illisible
est une dette silencieuse.

Dans SAHADIA :
- un code clair est toujours préféré à un code compact,
- une intention explicite est préférée à une astuce,
- la lisibilité est une forme de respect.

Un programme écrit aujourd’hui
doit pouvoir être relu demain
sans douleur.

---

### 3. Programmer comme on parle

SAHADIA assume une idée radicale :
> *le code peut ressembler à une phrase.*

Les mots-clés sont choisis pour leur sens,
pas pour leur brièveté.

Un programme SAHADIA doit pouvoir être lu à voix haute
et rester compréhensible.

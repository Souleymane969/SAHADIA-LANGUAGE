# Chapitre 14  
## Tests — Programmer avec confiance  
*Vérifier, comprendre, avancer sereinement*

Écrire un programme,
c’est formuler une intention.

Tester un programme,
c’est vérifier
que cette intention est respectée.

Les tests ne sont pas là
pour piéger,
mais pour rassurer.

---

## 1. La philosophie des tests en SAHADIA

Dans SAHADIA :
- un test est lisible
- un test explique ce qu’il vérifie
- un test échoue calmement

Un test est un dialogue
avec votre code.

---

## 2. Qu’est-ce qu’un test ?

Un test est un petit programme
qui vérifie un comportement précis.

Il pose une question simple :
> *Est-ce que ce code fait bien ce que j’attends ?*

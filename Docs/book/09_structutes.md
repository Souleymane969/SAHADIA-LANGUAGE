# Chapitre 9  
## Structures — Donner une forme aux données  
*Objets simples, champs nommés, lisibilité*

Jusqu’ici, nous avons manipulé des valeurs,
puis des listes de valeurs.

Mais certaines données vont naturellement ensemble.

Un utilisateur a :
- un nom
- un âge
- un email

Ces informations ne sont pas une liste.
Elles forment une **structure**.

---

## 1. Qu’est-ce qu’une structure ?

Une structure est :
> **un ensemble de champs nommés regroupés sous une seule entité**

Au lieu de multiplier les variables,
on donne une forme claire aux données.

---

## 2. La philosophie des structures en SAHADIA

Dans SAHADIA :
- une structure est lisible
- ses champs sont explicites
- l’accès aux données est naturel

Une structure n’est pas un objet complexe.
C’est une **carte de sens**.
